# Authentication

Fizzy MCP uses Personal Access Tokens (PATs) to authenticate with the Fizzy API. The CLI provides a streamlined authentication flow that automatically configures your AI agents.

## Deployment Modes

Fizzy MCP supports two deployment modes:

<CardGrid>
  <Card title="Remote Mode (Recommended)" icon="rocket">
    Uses our hosted service for zero-maintenance operation. No local server process required.
    **Endpoint:** `https://fizzy-mcp-hosted.ryanyogan.workers.dev/mcp`
  </Card>
  <Card title="Local Mode" icon="laptop">
    Traditional stdio MCP server running locally via `npx`. Full control over the server process.
  </Card>
</CardGrid>

<Aside type="tip">
  **Remote mode is recommended** for most users. It eliminates the need to manage a local server
  process and provides automatic updates. Local mode is available for users who prefer full control
  or have specific network requirements.
</Aside>

## Getting a Personal Access Token

<Steps>

1. **Open Fizzy**

   Go to [app.fizzy.do](https://app.fizzy.do) and sign in to your account.

2. **Navigate to API Settings**

   Click your avatar in the top-right corner, then go to **Profile** → **API**.

   Or navigate directly to: [https://app.fizzy.do/my/profile/api](https://app.fizzy.do/my/profile/api)

3. **Create a New Token**

   Click **Create Token** and give it a descriptive name (e.g., "Claude Desktop MCP").

4. **Copy the Token**

   Copy the generated token. It starts with `fz_` and won't be shown again.

</Steps>

<Aside type="caution">
  Store your token securely! It provides full access to your Fizzy account and cannot be retrieved
  after creation.
</Aside>

## Running Authentication

Run the auth command to configure Fizzy MCP:

```bash
npx @fizzy-mcp/server auth
```

### CLI Options

The `auth` command supports the following options:

| Option                | Description                                       |
| --------------------- | ------------------------------------------------- |
| `-m, --mode <mode>`   | Deployment mode: `remote` (default) or `local`    |
| `-k, --api-key <key>` | API key for higher rate limits (remote mode only) |
| `--no-configure`      | Skip auto-configuration of AI agents              |

**Examples:**

```bash
# Interactive auth (recommended)
npx @fizzy-mcp/server auth

# Explicitly use remote mode
npx @fizzy-mcp/server auth --mode remote

# Use local mode
npx @fizzy-mcp/server auth --mode local

# Remote mode with API key for higher rate limits
npx @fizzy-mcp/server auth --mode remote --api-key fmcp_xxxxxxxxxxxxx

# Auth without configuring AI agents
npx @fizzy-mcp/server auth --no-configure
```

## Rate Limits (Remote Mode)

Remote mode enforces rate limits to ensure fair usage:

| Tier         | Write Operations | Read Operations |
| ------------ | ---------------- | --------------- |
| Anonymous    | 100/day          | Unlimited       |
| With API Key | 1,000/day        | Unlimited       |

<Aside type="tip">
  Write operations include creating, updating, deleting, and moving cards. Read operations (listing
  boards, viewing cards, etc.) are unlimited.
</Aside>

To upgrade your rate limits, add an API key:

```bash
npx @fizzy-mcp/server auth --api-key fmcp_xxxxxxxxxxxxx
```

Or reconfigure:

```bash
npx @fizzy-mcp/server auth --mode remote
```

## Token Security

Fizzy MCP stores your credentials securely:

- **File permissions**: Config file is created with mode `600` (owner-only read/write)
- **Hidden input**: Token is not displayed when typing
- **No logging**: Tokens are never logged or transmitted except to the Fizzy API
- **Secure storage**: API keys are stored separately from access tokens

### Where Credentials Are Stored

| Platform    | Location                                      |
| ----------- | --------------------------------------------- |
| macOS/Linux | `~/.config/fizzy-mcp/config.json`             |
| Windows     | `%USERPROFILE%\.config\fizzy-mcp\config.json` |

### Config File Structure

```json
{
  "mode": "remote",
  "accessToken": "fz_xxxxxxxxxxxxx",
  "apiKey": "fmcp_xxxxxxxxxxxxx",
  "accountSlug": "/ryan"
}
```

## Revoking Tokens

To revoke a token:

1. Go to [app.fizzy.do](https://app.fizzy.do)
2. Navigate to **Profile** → **API**
3. Find the token and click **Revoke**

Then clear local credentials:

```bash
npx @fizzy-mcp/server logout
```

## Using Environment Variables

For CI/CD or server deployments, use environment variables:

```bash
# Required
export FIZZY_ACCESS_TOKEN=fz_xxxxxxxxxxxxx

# Optional
export FIZZY_ACCOUNT_SLUG=/my-org
export FIZZY_MODE=remote
export FIZZY_API_KEY=fmcp_xxxxxxxxxxxxx
```

These take precedence over the config file.

### Environment Variable Reference

| Variable             | Description                           | Example              |
| -------------------- | ------------------------------------- | -------------------- |
| `FIZZY_ACCESS_TOKEN` | Personal Access Token (required)      | `fz_xxxxxxxxxxxxx`   |
| `FIZZY_ACCOUNT_SLUG` | Account to use                        | `/my-org`            |
| `FIZZY_MODE`         | Deployment mode                       | `remote` or `local`  |
| `FIZZY_API_KEY`      | API key for rate limits (remote only) | `fmcp_xxxxxxxxxxxxx` |

## Troubleshooting

### "Invalid access token"

- Ensure the token is copied correctly (no extra spaces)
- Check that the token hasn't been revoked
- Verify you're using a PAT (prefix `fz_`), not an API key (prefix `fmcp_`)

### "Invalid API key"

- Ensure the API key starts with `fmcp_`
- Check that the key hasn't been revoked
- Note: API keys are only used in remote mode

### "Rate limit exceeded"

- Add an API key to increase your limit to 1,000 writes/day
- Wait until the rate limit resets (daily)
- Consider switching to local mode for unlimited writes

### "No accessible accounts"

- Your token may not have access to any accounts
- Contact your Fizzy administrator to grant access

### "Authentication failed"

- Check your internet connection
- Verify the Fizzy API is accessible: `curl https://app.fizzy.do/api/health`
- For remote mode, verify the hosted service: `curl https://fizzy-mcp-hosted.ryanyogan.workers.dev/health`

### Browser doesn't open

If pressing Enter doesn't open your browser:

1. Manually navigate to [https://app.fizzy.do/my/profile/api](https://app.fizzy.do/my/profile/api)
2. Copy your token
3. Paste it at the prompt
