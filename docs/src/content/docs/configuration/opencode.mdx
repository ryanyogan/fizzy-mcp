# OpenCode Configuration

This guide covers setting up Fizzy MCP with [OpenCode](https://opencode.ai), the open-source AI coding assistant.

<Aside type="caution">
  **OpenCode uses a different configuration format** than Claude Desktop and Cursor. Pay attention
  to the format differences documented below.
</Aside>

## Deployment Modes

Fizzy MCP supports two deployment modes:

| Mode                     | Description                                   | Best For                                        |
| ------------------------ | --------------------------------------------- | ----------------------------------------------- |
| **Remote** (Recommended) | Connects to hosted Fizzy MCP service via HTTP | Most users, zero maintenance, always up-to-date |
| **Local**                | Runs MCP server locally via npx               | Offline use, custom modifications, self-hosting |

<Aside type="tip">
  **Remote mode is recommended** for most users. It requires no local installation, receives
  automatic updates, and provides the best reliability.
</Aside>

## Auto-Configuration

The easiest way to configure OpenCode is through the Fizzy MCP CLI:

```bash
npx fizzy-do-mcp auth
```

Or use the configure command directly:

```bash
npx fizzy-do-mcp configure
```

When prompted, select OpenCode for automatic configuration. The CLI will handle the format differences automatically.

## Configuration File Location

OpenCode configuration is at:

```
~/.config/opencode/opencode.json
```

## Manual Configuration

Choose your deployment mode. Note that OpenCode uses `type` instead of `transport`:

<Tabs>
<TabItem label="Remote Mode (Recommended)">

Remote mode connects to the hosted Fizzy MCP service via HTTP.

### Basic Remote Configuration

```json
{
  "mcp": {
    "fizzy": {
      "type": "http",
      "url": "https://fizzy-mcp-hosted.ryanyogan.workers.dev/mcp",
      "headers": {
        "X-Fizzy-Token": "your-fizzy-token"
      }
    }
  }
}
```

### Remote Configuration with All Headers

```json
{
  "mcp": {
    "fizzy": {
      "type": "http",
      "url": "https://fizzy-mcp-hosted.ryanyogan.workers.dev/mcp",
      "headers": {
        "X-Fizzy-Token": "your-fizzy-token",
        "X-Fizzy-Account-Slug": "/my-org",
        "X-API-Key": "fmcp_your_api_key"
      }
    }
  }
}
```

</TabItem>
<TabItem label="Local Mode">

Local mode runs the MCP server on your machine.

### Basic Local Configuration

```json
{
  "mcp": {
    "fizzy": {
      "type": "local",
      "command": ["npx", "-y", "fizzy-do-mcp@latest"]
    }
  }
}
```

### Local Configuration with Environment Variables

```json
{
  "mcp": {
    "fizzy": {
      "type": "local",
      "command": ["npx", "-y", "fizzy-do-mcp@latest"],
      "env": {
        "FIZZY_ACCESS_TOKEN": "your-fizzy-token",
        "FIZZY_ACCOUNT_SLUG": "/my-org"
      }
    }
  }
}
```

</TabItem>
</Tabs>

After adding the configuration, reload or restart OpenCode to apply the changes.

## OpenCode-Specific Format

<Aside type="caution">
  OpenCode's MCP configuration differs significantly from Claude Desktop and Cursor. Make sure you
  use the correct format!
</Aside>

### Key Differences

| Feature           | OpenCode                      | Claude Desktop / Cursor                                     |
| ----------------- | ----------------------------- | ----------------------------------------------------------- |
| Config key        | `mcp`                         | `mcpServers`                                                |
| Remote type field | `"type": "http"`              | `"transport": "http"`                                       |
| Local type field  | `"type": "local"`             | Not used                                                    |
| Command format    | Array: `["npx", "-y", "pkg"]` | String + args: `"command": "npx"` + `"args": ["-y", "pkg"]` |

## Remote Mode Headers

When using remote mode, the following headers are available:

| Header                 | Required | Description                                              |
| ---------------------- | -------- | -------------------------------------------------------- |
| `X-Fizzy-Token`        | **Yes**  | Your Fizzy personal access token (from app.fizzy.do)     |
| `X-Fizzy-Account-Slug` | No       | Account slug like `/my-org` to select a specific account |
| `X-API-Key`            | No       | API key for higher rate limits (prefix: `fmcp_`)         |

<Aside>
  To get your Fizzy token, visit [app.fizzy.do](https://app.fizzy.do), go to Settings > Personal Access Tokens, and create a new token.
</Aside>

## Verifying the Connection

After configuration:

1. Start OpenCode
2. Check the MCP status indicator

## Authentication for Local Mode

After configuring OpenCode with local mode, authenticate with Fizzy:

```bash
npx fizzy-do-mcp auth
```

This stores your credentials securely in `~/.config/fizzy-mcp/config.json` (mode 0600). No need to put tokens in your MCP config unless you prefer explicit configuration.

## Troubleshooting

### Server not connecting

1. Verify the config file location (`~/.config/opencode/opencode.json`)
2. Check that you're using `type` not `transport`
3. For local mode, ensure `type: "local"` is present
4. For remote mode, ensure `type: "http"` is present
5. Test server manually (local mode): `npx fizzy-do-mcp status`

### "Not authenticated" errors (Remote Mode)

Verify your `X-Fizzy-Token` header is correct:

1. Check that the token is valid at [app.fizzy.do](https://app.fizzy.do)
2. Ensure there are no extra spaces or quotes around the token
3. Regenerate the token if needed

### "Not authenticated" errors (Local Mode)

Run `fizzy-mcp auth` to set up or refresh your credentials:

```bash
npx fizzy-do-mcp auth
```

### Invalid configuration format

OpenCode requires specific format. Common mistakes:

- Using `mcpServers` instead of `mcp`
- Using `transport` instead of `type`
- Using separate `command` and `args` instead of a single `command` array

### Connection issues (Remote Mode)

1. Check your internet connection
2. Verify the hosted service is accessible:
   ```bash
   curl https://fizzy-mcp-hosted.ryanyogan.workers.dev/health
   ```
3. If using a corporate network, check for firewall rules

## Best Practices

1. **Use remote mode** for automatic updates and zero maintenance
2. **Double-check the format** - OpenCode uses different syntax than other agents
3. **Keep tokens secure** - never commit config files with tokens to version control
4. **Use `X-Fizzy-Account-Slug`** if you have access to multiple Fizzy accounts
5. **Use the CLI** for auto-configuration to avoid format mistakes
