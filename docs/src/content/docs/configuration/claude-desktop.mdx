# Claude Desktop Configuration

This guide covers setting up Fizzy MCP with [Claude Desktop](https://claude.ai/download), Anthropic's official desktop application.

## Deployment Modes

Fizzy MCP supports two deployment modes:

| Mode                     | Description                                   | Best For                                        |
| ------------------------ | --------------------------------------------- | ----------------------------------------------- |
| **Remote** (Recommended) | Connects to hosted Fizzy MCP service via HTTP | Most users, zero maintenance, always up-to-date |
| **Local**                | Runs MCP server locally via npx               | Offline use, custom modifications, self-hosting |

<Aside type="tip">
  **Remote mode is recommended** for most users. It requires no local installation, receives
  automatic updates, and provides the best reliability.
</Aside>

## Auto-Configuration

The easiest way to configure Claude Desktop is through the Fizzy MCP CLI:

```bash
npx fizzy-do-mcp auth
```

Or use the configure command directly:

```bash
npx fizzy-do-mcp configure
```

When prompted, select Claude Desktop for automatic configuration. The CLI will detect the best mode for your setup.

## Manual Configuration

Choose your deployment mode and add the configuration to your Claude Desktop config file:

<Tabs>
<TabItem label="Remote Mode (Recommended)">

Remote mode connects to the hosted Fizzy MCP service via HTTP transport.

### Basic Remote Configuration

```json
{
  "mcpServers": {
    "fizzy": {
      "transport": "http",
      "url": "https://fizzy-mcp-hosted.ryanyogan.workers.dev/mcp",
      "headers": {
        "X-Fizzy-Token": "your-fizzy-token"
      }
    }
  }
}
```

### Remote Configuration with All Headers

```json
{
  "mcpServers": {
    "fizzy": {
      "transport": "http",
      "url": "https://fizzy-mcp-hosted.ryanyogan.workers.dev/mcp",
      "headers": {
        "X-Fizzy-Token": "your-fizzy-token",
        "X-Fizzy-Account-Slug": "/my-org",
        "X-API-Key": "fmcp_your_api_key"
      }
    }
  }
}
```

</TabItem>
<TabItem label="Local Mode">

Local mode runs the MCP server on your machine via stdio.

### Basic Local Configuration

```json
{
  "mcpServers": {
    "fizzy": {
      "command": "npx",
      "args": ["-y", "fizzy-do-mcp@latest"]
    }
  }
}
```

### Local Configuration with Environment Variables

```json
{
  "mcpServers": {
    "fizzy": {
      "command": "npx",
      "args": ["-y", "fizzy-do-mcp@latest"],
      "env": {
        "FIZZY_ACCESS_TOKEN": "your-fizzy-token",
        "FIZZY_ACCOUNT_SLUG": "/my-org"
      }
    }
  }
}
```

</TabItem>
</Tabs>

After adding the configuration, completely close and reopen Claude Desktop to load the new settings.

## Remote Mode Headers

When using remote mode, the following headers are available:

| Header                 | Required | Description                                              |
| ---------------------- | -------- | -------------------------------------------------------- |
| `X-Fizzy-Token`        | **Yes**  | Your Fizzy personal access token (from app.fizzy.do)     |
| `X-Fizzy-Account-Slug` | No       | Account slug like `/my-org` to select a specific account |
| `X-API-Key`            | No       | API key for higher rate limits (prefix: `fmcp_`)         |

<Aside>
  To get your Fizzy token, visit [app.fizzy.do](https://app.fizzy.do), go to Settings > Personal Access Tokens, and create a new token.
</Aside>

### Getting an API Key

API keys provide higher rate limits for power users. To obtain one:

1. Contact Fizzy support or visit the API dashboard
2. API keys are prefixed with `fmcp_` (e.g., `fmcp_abc123xyz...`)
3. Add the key to your configuration's `X-API-Key` header

## Verifying the Connection

After restarting Claude Desktop:

1. Start a new conversation
2. Look for the MCP tools icon (usually a wrench or plugin icon)
3. You should see "fizzy" listed as a connected server

Try asking Claude:

> "What Fizzy boards do I have access to?"

## Configuration Options

### Using a Global Installation (Local Mode Only)

If you installed Fizzy MCP globally:

```json
{
  "mcpServers": {
    "fizzy": {
      "command": "fizzy-mcp"
    }
  }
}
```

### Authentication for Local Mode

When using local mode, authenticate with Fizzy:

```bash
npx fizzy-do-mcp auth
```

This stores your credentials securely in `~/.config/fizzy-mcp/config.json` (mode 0600). No need to put tokens in your MCP config unless you prefer explicit configuration.

## Troubleshooting

### Tools not appearing

1. Check that the config file is valid JSON
2. Verify the file is in the correct location
3. Ensure you've restarted Claude Desktop completely
4. Check Claude Desktop logs for MCP connection errors

### "Not authenticated" errors (Remote Mode)

Verify your `X-Fizzy-Token` header is correct:

1. Check that the token is valid at [app.fizzy.do](https://app.fizzy.do)
2. Ensure there are no extra spaces or quotes around the token
3. Regenerate the token if needed

### "Not authenticated" errors (Local Mode)

Run `fizzy-mcp auth` to set up or refresh your credentials:

```bash
npx fizzy-do-mcp auth
```

### Connection timeouts (Remote Mode)

1. Check your internet connection
2. Verify the hosted service is accessible: `curl https://fizzy-mcp-hosted.ryanyogan.workers.dev/health`
3. If using a corporate network, check for firewall rules blocking the connection

### Connection timeouts (Local Mode)

1. Check your internet connection
2. Verify Fizzy API is accessible: `curl https://app.fizzy.do/api/health`
3. Try increasing the timeout in Claude Desktop settings

### Server crashes (Local Mode)

Check the server logs. You can run the server manually to see errors:

```bash
npx fizzy-do-mcp
```

## Best Practices

1. **Use remote mode** for automatic updates and zero maintenance
2. **Keep tokens secure** - never commit them to version control
3. **Use `X-Fizzy-Account-Slug`** if you have access to multiple accounts
4. **Restart Claude Desktop** after any configuration changes
5. **Check `fizzy-mcp status`** if you encounter authentication issues (local mode)
