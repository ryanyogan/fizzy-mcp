# Fizzy MCP Hosted Service

The Fizzy MCP Hosted Service provides **remote MCP access** without requiring you to run a local server. Your AI assistant connects directly to our cloud infrastructure, giving you instant access to all Fizzy MCP tools from anywhere.

## What is the Hosted Service?

The hosted service is a fully managed MCP server running on **Cloudflare's global edge network**. Instead of running `npx @fizzy-mcp/server` locally, your AI agent connects via HTTP to our hosted endpoint:

```
https://fizzy-mcp-hosted.ryanyogan.workers.dev/mcp
```

This endpoint speaks the Model Context Protocol over **HTTP Streamable transport**, making it compatible with any MCP client that supports HTTP-based connections.

## Authentication

The hosted service requires your **Fizzy Personal Access Token** to authenticate requests to the Fizzy API. This token is passed via the `X-Fizzy-Token` HTTP header.

### Required Headers

| Header          | Required | Description                                                   |
| --------------- | -------- | ------------------------------------------------------------- |
| `X-Fizzy-Token` | Yes      | Your Fizzy Personal Access Token (starts with `fz_`)          |
| `X-API-Key`     | No       | Optional API key for higher rate limits (starts with `fmcp_`) |
| `Content-Type`  | Yes      | Must be `application/json`                                    |

<Aside type="caution">
  Your Fizzy token is sent to the hosted service with each request. The service uses it only to
  authenticate with the Fizzy API on your behalf and does not store it permanently.
</Aside>

### How Authentication Works

<Steps>

1. **You configure your AI agent** with your Fizzy token in the headers

2. **Your AI agent sends an MCP request** to the hosted service

3. **The hosted service validates** the request format and checks rate limits

4. **The hosted service forwards** the request to the Fizzy API using your token

5. **The Fizzy API returns** the response, which is relayed back to your AI agent

</Steps>

## Quick Start

Get up and running with the hosted service in minutes:

<Steps>

1.  **Get your Fizzy token**

    Go to [app.fizzy.do](https://app.fizzy.do), navigate to **Profile** → **API**, and create a Personal Access Token.

2.  **Test the connection**

    Use curl to verify your token works:

    ```bash
    curl -X POST https://fizzy-mcp-hosted.ryanyogan.workers.dev/mcp \
      -H "Content-Type: application/json" \
      -H "X-Fizzy-Token: fz_your_token_here" \
      -d '{
        "jsonrpc": "2.0",
        "id": 1,
        "method": "tools/list",
        "params": {}
      }'
    ```

    You should receive a JSON response listing all available tools.

3.  **Configure your AI agent** (see examples below)

4.  **Start using Fizzy**

    Open your AI agent and try a command like:

    > "List all my Fizzy boards"

</Steps>

## Example: Calling a Tool

Here's a complete example of calling the `fizzy_list_boards` tool via the hosted service:

```bash
curl -X POST https://fizzy-mcp-hosted.ryanyogan.workers.dev/mcp \
  -H "Content-Type: application/json" \
  -H "X-Fizzy-Token: fz_your_token_here" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "tools/call",
    "params": {
      "name": "fizzy_list_boards",
      "arguments": {}
    }
  }'
```

## Comparison: Hosted vs Local

| Feature            | Hosted Service                     | Local Server                  |
| ------------------ | ---------------------------------- | ----------------------------- |
| **Setup**          | None (just configure headers)      | Install Node.js, run `npx`    |
| **Updates**        | Automatic                          | Manual (`npm update`)         |
| **Rate Limits**    | Yes (100-1,000 writes/day)         | None                          |
| **Offline Access** | No                                 | Yes (after auth)              |
| **Token Storage**  | Sent with each request             | Stored locally                |
| **Latency**        | Network roundtrip                  | Direct stdio                  |
| **Best For**       | Remote dev, cloud IDEs, simplicity | High-volume, offline, privacy |

## Security Considerations

### Token Handling

- Your Fizzy token is transmitted over HTTPS (TLS 1.3)
- The token is used only for the duration of the request
- Tokens are never logged or stored permanently
- Each request is isolated at the edge

### Network Security

- All connections use HTTPS with modern cipher suites
- Cloudflare provides DDoS protection and WAF
- Edge locations reduce attack surface
- Request validation prevents injection attacks

### Best Practices

<Aside type="tip">
  - Rotate your Fizzy token periodically - Use an API key for production workflows - Monitor your
  usage via rate limit headers - Never commit tokens to version control
</Aside>

## Troubleshooting

### "401 Unauthorized"

- Verify your `X-Fizzy-Token` header is set correctly
- Ensure the token starts with `fz_`
- Check that the token hasn't been revoked in Fizzy

### "429 Too Many Requests"

- You've exceeded your rate limit
- Check the `X-RateLimit-Reset` header for when limits reset
- Consider getting an API key for higher limits

### "Connection refused" or timeout

- Verify you have internet connectivity
- Check if Cloudflare is experiencing issues: [cloudflarestatus.com](https://www.cloudflarestatus.com)
- Ensure your firewall allows HTTPS to `*.workers.dev`
